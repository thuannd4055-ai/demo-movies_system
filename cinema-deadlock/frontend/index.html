✅ Phương pháp 1: Ordered Locking
Luôn đặt khóa theo thứ tự tăng dần của seat_id. T1 và T2 đều khóa seat nhỏ trước → không tạo vòng tròn chờ → không deadlock.
-- Sắp xếp seat IDs trước khi khóa seat_ids = sorted([seat_id1, seat_id2]) -- [3, 7] -- T1 và T2 đều làm theo thứ tự: SELECT * FROM seats WHERE id = 3 FOR UPDATE; SELECT * FROM seats WHERE id = 7 FOR UPDATE;
⏰ Phương pháp 2: Lock Timeout
Cấu hình lock_timeout và deadlock_timeout để PostgreSQL tự phát hiện và rollback victim.
-- Cấu hình timeout (giảm thời gian detect) SET lock_timeout = '5s'; SET deadlock_timeout = '1s'; -- PostgreSQL sẽ tự rollback victim sau 1-2s
🔄 Phương pháp 3: 2PL (Two-Phase Locking)
Xin tất cả khóa trước khi thực hiện bất kỳ thao tác nào. Không bao giờ xin thêm khóa sau khi đã giải phóng một khóa.
BEGIN; -- Phase 1 (Growing): Xin tất cả khóa ngay SELECT * FROM seats WHERE id IN (3,7) FOR UPDATE; -- Xin cả 2 khóa cùng lúc! -- Phase 2 (Shrinking): Thực hiện rồi giải phóng UPDATE seats SET status='booked' WHERE id IN (3,7); COMMIT;
